What is Auto?

Auto is a small Windows tool that starts multiple programs or servers for you from a single command. You list what you want to run in a simple config file; Auto opens each one in its own window (or in the background). No more opening five terminals by hand every morning—one command and you’re ready to work.

Why use Auto?

Spend less time on repetitive setup and more time building.

  • Save time — Start your full dev environment with one command instead of many.
  • Stay consistent — Same setup every time: frontend, backend, database, watchers—all in one place.
  • Keep things visible — Each app runs in its own window so you can see logs and output where you need them.
  • Simple to share — One config file in your project; teammates run the same command and get the same environment.
  • Free and open source — No account, no subscription. Download and use.

When to use Auto?

Perfect for everyday development and any workflow that needs several things running at once.

Daily development

You work on a project that needs a frontend server, a backend API, a database, and maybe a file watcher. With Auto, one command starts them all in separate windows with clear labels.

Running multiple servers

Microservices, APIs, or multiple apps that need to run together. Define them once in a config file and launch everything with a single auto dev (or any name you choose).

Background services

Some tasks (e.g. killing old processes, starting a local DB) are better run in the background or in a minimized window. Auto can do that so your main terminals stay clean.

Onboarding and demos

New teammates or a quick demo? They run one command and get the same environment. No “run these five commands in order” instructions.

Advantages at a glance

  • One command starts everything you need.
  • Each app in its own window—easy to watch and debug.
  • Simple text config: no GUI to learn, easy to version in Git.
  • Works with Windows Terminal and supports custom tab titles.
  • Optional background or minimized windows for services you don’t need to see.
  • Lightweight, no runtime or account required.

What you can do
